<dl class="py class">
<dt class="sig sig-object py" id="great_expectations.expectations.metrics.metric_provider.MetricProvider">
<em class="property"><span class="pre">class</span><span class="w"> </span></em><span class="sig-prename descclassname"><span class="pre">great_expectations.expectations.metrics.metric_provider.</span></span><span class="sig-name descname"><span class="pre">MetricProvider</span></span><a class="headerlink" href="#great_expectations.expectations.metrics.metric_provider.MetricProvider" title="Permalink to this definition">#</a></dt>
<dd><p>Base class for all metric providers.</p>
<dl class="simple">
<dt>MetricProvider classes <em>must</em> have the following attributes set:</dt><dd><ol class="arabic simple">
<li><p><cite>metric_name</cite>: the name to use. Metric Name must be globally unique in
a great_expectations installation.</p></li>
<li><p><cite>domain_keys</cite>: a tuple of the <em>keys</em> used to determine the domain of the
metric</p></li>
<li><p><cite>value_keys</cite>: a tuple of the <em>keys</em> used to determine the value of
the metric.</p></li>
</ol>
</dd>
</dl>
<p>In some cases, subclasses of Expectation, such as TableMetricProvider will already
have correct values that may simply be inherited.</p>
<dl class="simple">
<dt>MetricProvider classes <em>must</em> implement the following:</dt><dd><p>1. <cite>_get_evaluation_dependencies</cite>. Note that often, _get_evaluation_dependencies should
augment dependencies provided by a parent class; consider calling super()._get_evaluation_dependencies</p>
</dd>
</dl>
<p>In some cases, subclasses of Expectation, such as MapMetricProvider will already
have correct implementations that may simply be inherited.</p>
<dl class="simple">
<dt>Additionally, they <em>may</em> provide implementations of:</dt><dd><p>1. Data Docs rendering methods decorated with the @renderer decorator. See the guide
“How to create renderers for custom expectations” for more information.</p>
</dd>
</dl>
</dd></dl>
