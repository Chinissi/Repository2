<p>An interface for configuring Metrics.</p>
<p>MetricConfiguration allows the configuration of domain information, dependencies and additional metric-specific
configurations.  Metrics are computed attributes of data, and are derived from one or more Batches that can then
be used to evaluate Expectations or to summarize the result of the Validation.</p>
<dl class="field-list simple">
<dt class="field-odd">Parameters</dt>
<dd class="field-odd"><ul class="simple">
<li><p><strong>metric_name</strong> (<em>str</em>) – name of the Metric defined by the current MetricConfiguration.</p></li>
<li><p><strong>metric_domain_kwargs</strong> (<em>dict</em>) – provides information on where the Metric can be calculated. For instance, a
MapCondition metric can include the name of the column that the Metric is going to be run on.</p></li>
<li><p><strong>metric_value_kwargs</strong> (<em>optional</em><em>[</em><em>dict</em><em>]</em>) – Optional kwargs that define values specific to each Metric.  For instance,
a Metric that partitions a column can define the method of partitioning (<cite>uniform</cite> bins) and the number of
bins (<cite>n_bins</cite>) as <cite>metric_value_kwargs</cite>.</p></li>
</ul>
